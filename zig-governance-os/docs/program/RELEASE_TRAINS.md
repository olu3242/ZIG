# Program Release Trains

## Purpose

Program release trains convert the architecture into scheduled delivery lanes.

## Train 1: MVP Governance Platform

Target: first paying customers.

Scope:

- Data
- Identity
- Persona
- Graph
- Knowledge
- Data Fabric
- Workflow
- Runtime
- Automation
- Controls
- Evidence
- Assessment & Audit
- Score
- Mission Control
- Analytics

Exit criteria:

- Customer can create tenant, create project, select framework, manage controls, upload evidence, run assessment, view governance score, and generate report.

## Train 2: Learning And AI Platform

Scope:

- Learning OS
- Scenario Lab
- Practice Lab
- AI Coach
- Digital Assistants
- AI Command Center

Exit criteria:

- User can learn, practice, assess, and receive AI-guided governance support.

## Train 3: Certification And Marketplace

Scope:

- Certification Readiness
- Zig Governance Standard
- Credential Registry
- Marketplace
- Governance Content OS

Exit criteria:

- User can certify, verify credentials, and install governed content.

## Train 4: Governance Cloud

Scope:

- Governance Network
- Governance Exchange
- Trust Graph
- Autonomous Governance Cloud
- Platform certification and scale

Exit criteria:

- Organizations, consultants, auditors, students, employers, and certification bodies operate in Zig.
