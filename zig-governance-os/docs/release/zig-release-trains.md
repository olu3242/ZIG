# Zig Release Trains

## Purpose

Zig now has enough implementation scope that the roadmap must move from an endless batch list to release trains. Release trains keep development focused on customer outcomes instead of module accumulation.

The long-term evolution is:

```text
Zig Platform -> Zig Ecosystem -> Zig Network -> Zig Standard
```

## Immediate Critical Path

Do not start ecosystem or network batches until the technical foundation is real.

The immediate coding focus remains:

```text
Batch 21 -> Core Data Platform
Batch 22 -> Identity Platform
Batch 23 -> Governance Graph
Batch 24 -> Digital Governance Twin
```

These batches create the foundation required for every later capability:

- Persistent data model
- Authentication and tenant onboarding
- Relationship and traceability graph
- Simulation and forecasting layer

## Release Train 1: Zig Platform

Goal: make Zig a deployable governance operating platform.

Primary batches:

- Batch 21 Core Data Platform
