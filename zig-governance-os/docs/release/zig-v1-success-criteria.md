# Zig v1 Success Criteria

## Definition of Success

Zig v1 succeeds when a customer can learn governance, implement governance, operate governance, audit governance, certify governance, and improve governance without leaving Zig.

## Platform Criteria

- Every operational record is tenant-scoped.
- Every project record connects to the Governance Graph.
- Every module can be reached from the primary navigation or an in-context graph relationship.
- Every screen has populated, generated, or guided content.
- Every workflow creates, updates, resolves, or reports graph state.

## Governance Criteria

- A customer can create a project for at least one supported framework.
- The project can generate assets, risks, controls, evidence requirements, tasks, and assessments.
- Risks connect to assets and controls.
- Controls connect to frameworks and evidence.
- Evidence connects to audit and assessment workflows.
- Governance Score is visible, explainable, and affected by real actions.
- Recommendations include severity, reason, action, confidence, and source records.

## Learning And Certification Criteria

- Learning content is connected to operational gaps.
- Scenario Lab can simulate governance work and score outcomes.
- Assessments can evaluate user or project readiness.
- Certification pathways combine learning, labs, scenarios, assessments, and capstones.
- Credentials are backed by evidence and can be verified.

## AI Criteria

- AI Coach answers using tenant/project context.
- AI Command can generate governed draft outputs.
- Every AI output is explainable and permission-safe.
- Material AI actions require approval before official graph writes.
- AI recommendations can become tasks and be tracked to completion.

## Reporting Criteria

- Executive summaries are generated from graph data.
- Audit and readiness reports include traceable evidence.
- Board reporting can summarize score, risks, controls, evidence, and readiness.
- Export paths are available for launch-critical report types.

## Quality Criteria

- Core workflows pass integration tests.
- Tenant isolation is verified.
- RBAC behavior is verified for supported roles.
- Accessibility floor is met for primary screens.
- Performance is acceptable for realistic demo datasets.
- Production readiness report is complete.

## v1 Customer Outcome

A new customer can complete this journey:

```text
Create tenant -> Create project -> Select framework -> Generate program -> Review assets -> Assess risks -> Implement controls -> Attach evidence -> Complete assessment -> Review score -> Act on recommendations -> Generate report -> Continue learning or certification
```

without a blank screen, orphaned record, or unexplained recommendation.
