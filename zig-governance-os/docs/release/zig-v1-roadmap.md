# Zig v1 Roadmap

## Purpose

The v1 roadmap defines how Zig moves from implementation batches into a certified Governance Operating System. Batches remain useful for sequencing, but the roadmap prevents the product from drifting into disconnected feature delivery.

## Roadmap Structure

```text
Batch 1-6   Foundation, identity, framework, project, asset, and risk engines
Batch 7-12  Controls, evidence, audit, scoring, mission control, and learning
Batch 13-20 Scenario Lab, AI, reporting, marketplace, hardening, enterprise, and v1 certification
Phase A     Platform Convergence
Phase B     Operational Intelligence
Phase C     Autonomous Governance
Phase D     Certification Ecosystem
Phase E     Governance Cloud
```

## v1 Release Principle

Zig v1 is not complete because every module exists. Zig v1 is complete when every module operates through one graph, one access model, one intelligence layer, and one measurable governance improvement loop.

## Release Gates

- Architecture gate: graph model and module contracts are consistent.
- Security gate: tenant isolation and RBAC are enforced.
- Quality gate: core workflows are tested.
- UX gate: no blank states or dead ends.
- Intelligence gate: recommendations are explainable.
- Certification gate: launch package and readiness reports are complete.
