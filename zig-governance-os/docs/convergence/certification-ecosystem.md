# Certification Ecosystem

## Purpose

The Certification Ecosystem turns Zig into a governance learning and credential platform, not just an operations platform. Certification paths combine learning, labs, scenarios, assessments, capstones, and operational governance practice.

## Certifications

- Zig GRC Analyst
- Zig Risk Analyst
- Zig Compliance Analyst
- Zig Internal Auditor
- Zig Governance Practitioner
- Zig Virtual CISO

## Certification Model

```text
Learning -> Labs -> Scenarios -> Assessments -> Capstone -> Certification
```

Each certification must be tied to practical capability, not passive content completion.

## Credential Objects

| Object | Description |
|---|---|
| Certification | Named credential with requirements and validity rules |
| Badge | Smaller credential for a skill, lab, scenario, or module |
| Capstone | Practical project or simulated governance program |
| Credential Record | User-owned award record with evidence, date, and status |
| Renewal Requirement | Continuing education or reassessment rule |

## Integration With Operations

Certification should reflect operational performance where appropriate:

- Risk Register performance can support Risk Analyst certification.
- Control Library performance can support Compliance Analyst certification.
- Audit findings and review quality can support Internal Auditor certification.
- Scenario Lab performance can support advanced credentials.
- AI Coach can recommend certification paths from role and graph gaps.

## Credential Registry

The registry must support:

- Credential definition
- Awarded credential records
- Badge metadata
- Verification status
- Expiration and renewal
- Capstone evidence

## Acceptance Criteria

- A user can progress from learning to lab to scenario to assessment to credential.
- Certification state is tenant-safe but portable enough for a user's professional profile.
- Credentials are backed by evidence and assessment outcomes.
- Learning recommendations can target a certification path.
- Certification readiness can be reported to users and tenant admins.
