# 03_BOARD_REPORTING_STRUCTURE

## Objectives
- Structure a board report: executive summary, score trend, top risks, recommendations, ask.
- Decide what to exclude from a board report that would belong in a compliance/audit report instead.
- Set a reporting cadence appropriate to governance program maturity.

## Business Context
This converts governance/risk literacy (Lessons 01-02) into the actual artifact a board
receives.

## Scenario Mapping
GovSec (`docs/scenarios/GOVSEC.md`) — build its first board-ready governance report.

## Framework Mapping
Trains on `GovernanceScore` + `Recommendation` records assembled into report structure.

## Diagram Requirements
- Board report structure diagram (section by section)
- Reporting cadence decision diagram (by program maturity)

## Knowledge Check
1. What belongs in a compliance report but should be excluded from a board report?
2. What determines an appropriate board reporting cadence?

## Artifact Produced
Board Report (draft) — see `docs/artifacts/` Board Report template.

## Visual Assets Required
- Board Report Structure Diagram
- Reporting Cadence Decision Diagram
