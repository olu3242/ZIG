# 02_VENDOR_DUE_DILIGENCE

## Objectives
- Run a due-diligence process before onboarding a new vendor asset.
- Determine the depth of diligence required based on data access and criticality.
- Identify red flags that should block or delay vendor onboarding.

## Business Context
Due diligence is the control point before a vendor ever touches data — this lesson trains
the gate, not just the ongoing monitoring covered later in the track.

## Scenario Mapping
RetailNova (`docs/scenarios/RETAILNOVA.md`) — due-diligence its informal vendor risk
process into a documented gate.

## Framework Mapping
Trains on `Asset.criticality` as the input that determines diligence depth.
