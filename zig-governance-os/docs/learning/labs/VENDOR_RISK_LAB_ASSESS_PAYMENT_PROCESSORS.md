# Vendor Risk Lab: Assess Payment Processor Vendors

## Scenario
RetailNova (`docs/scenarios/RETAILNOVA.md`).

## Inputs
- RetailNova's two payment-processor vendor assets
- Vendor Risk track Lessons 02-04 (Due Diligence, Questionnaires & Evidence, Scoring)

## Tasks
1. Run due diligence on both payment processors based on their data access level.
2. Design and "send" a vendor security questionnaire scoped to each.
3. Score each vendor's risk using the likelihood/impact model plus a data-access multiplier.
4. Rank both vendors and recommend a monitoring cadence for each.

## Deliverables
- Vendor Assessment for both processors: due diligence notes, questionnaire, risk score, monitoring cadence.

## Scoring Rubric
| Criterion | Weight |
|---|---|
| Due diligence depth matches each vendor's actual data access level | 25% |
| Questionnaire is scoped to the vendor, not generic | 20% |
| Risk score includes the data-access multiplier with rationale | 30% |
| Monitoring cadence recommendation matches the resulting risk tier | 25% |

## AI Feedback Rules
ZARA reviews as **Auditor**: challenges any risk score missing the data-access multiplier
rationale, and as **Mentor**: suggests a more appropriate monitoring cadence if the
recommended one doesn't match the assessed risk tier.
