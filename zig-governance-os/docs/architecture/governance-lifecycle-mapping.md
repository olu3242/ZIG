# Governance Lifecycle Mapping

## Lifecycle Contract

| Entity | CREATE | ASSESS | IMPROVE | REPORT |
| --- | --- | --- | --- | --- |
| Project | Project workspace | Project health | Program tasks | Project summary |
| Asset | Inventory item | Risk exposure input | Classification recommendations | Asset coverage |
| Control | Control library item | Effectiveness/readiness input | Remediation source | Control coverage |
| Risk | Defined in Stage 2 | Score and residual risk | Treatment tasks | Risk exposure |
| Framework Requirement | Metadata | Readiness and gaps | Gap tasks | Readiness report |
| Evidence | Proof object | Evidence coverage | Evidence request task | Audit package |
| Task | Remediation work | Task risk context | Completion | Open/closed task report |
| Scenario | Generated practice context | Scenario score | Improvement artifacts | Portfolio artifact |
| Report | Not applicable | Snapshot source | Improvement proof | Executive output |

## Stage Validation Gates

| Stage | Validation |
| --- | --- |
| CREATE | Project, asset, and control records exist and are related |
| ASSESS | Risk and readiness scores are explainable |
| IMPROVE | Recommendations create tasks |
| REPORT | Reports include source counts and lifecycle posture |
