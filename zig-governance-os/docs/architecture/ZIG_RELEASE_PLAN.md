# Zig Release Plan

## Purpose

This architecture-level release plan defines the path from current platform foundation to usable MVP, then expansion layers.

## Release 0: Execution-Controlled Foundation

Includes:

- Execution governance docs
- Release trains
- Gap analysis
- Gap closure plan
- Reference architecture
- Program management docs

Exit criteria:

- Implementation queue is stable.
- Architecture is complete enough to stop roadmap expansion.

## Release 1: MVP Governance Platform

Target outcome:

```text
Create Tenant -> Create Project -> Select Framework -> Manage Controls -> Upload Evidence -> Run Assessment -> View Governance Score -> Generate Report
```

Core batches:

```text
21A, 21B, 22, 22A, 22B, 22C, 23, 23A, 23B, 24A-24L, 25, 26, 27, 28, 29, 29A
```

Exit criteria:

- Tenant isolation is validated.
- Audit logging works.
- Core graph relationships exist.
- Controls, evidence, assessments, and governance score operate end to end.
- Mission Control gives the user actionable next steps.

## Release 2: Learning And AI Expansion

Includes:

- Learning OS
- Scenario Lab
- Practice Lab
- AI Coach
- Digital Assistants
- AI Command Center

Exit criteria:

- Users can learn, practice, assess, receive AI guidance, and produce artifacts.

## Release 3: Certification And Marketplace

Includes:

- Certification readiness
- Zig Governance Standard
- Credential registry
- Marketplace
- Governance Exchange

Exit criteria:

- Users can prepare for, issue, verify, and renew credentials.
- Governance content can be packaged, installed, licensed, and versioned.

## Release 4: Governance Cloud

Includes:

- Governance Network
- Trust Graph
- Enterprise API ecosystem
- Autonomous Governance Cloud
- Platform certification and scale

Exit criteria:

- Organizations, consultants, auditors, students, employers, and certification bodies interact in Zig.
