# Landing Page V4 Spec

Date: 2026-06-19  
Release: MVP Commercialization Positioning

## Objective

Position Zig as a clear Governance Operating System using only capabilities that are implemented or substantially present in the current repository.

The page must answer:

1. What is Zig?
2. Who is Zig for?
3. What problems does Zig solve?
4. What capabilities does Zig provide?
5. Why is Zig different?
6. Which package should a buyer choose?

## Page Structure

1. Hero
2. Five Core Capabilities
3. How Zig Works
4. Inside the Zig Workspace
5. Why Organizations Choose Zig
6. Who Zig Is For
7. Framework Coverage
8. Pricing
9. Customer Outcomes
10. Final CTA
11. Footer

## Positioning Rules

- Use current capability language.
- Avoid future-state autonomous claims.
- Avoid Digital Twin, predictive governance, self-healing agents, and autonomous workforce as primary claims.
- Explain AI as assisted workflow support, not autonomous compliance.
- Present frameworks as mappings inside the workspace, not separate products.
- Present pricing as three packages only.

## Visual System

- Dark Zig OS canvas using `bg-zinc-950`.
- Glassmorphic panels using `bg-zinc-900/50`, `border-zinc-800`, and `backdrop-blur-xl`.
- Branded Logo in header and product visual.
- Animated workspace visual showing Mission Control, Frameworks, Risk Workspace, Evidence Workspace, Learning Center, and Executive Reporting.
- Mobile-first responsive sections.

## Primary CTAs

- Start Free -> `/signup`
- Book Demo -> `mailto:sales@zig.systems?subject=Zig%20Demo`

## Success Criteria

A first-time visitor should understand within 30 seconds:

- Zig is a governance operating system.
- It is for governance, compliance, risk, audit, and advisory teams.
- It helps organize projects, frameworks, risk, evidence, learning, and reporting.
- Pricing has Starter, Professional, and Enterprise packages.
